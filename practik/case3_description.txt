Кейс-задача №3. Кластеризация сменных рапортов (NLP)

1. Постановка задачи:
Мастера смен ежедневно заполняют текстовые журналы дефектов оборудования в свободной форме (например: «заклинило подшипник на конвейере №2», «перегрев мотора»). Анализировать тысячи таких записей вручную слишком долго. Требуется автоматизировать классификацию текстовых логов для поиска самых частых причин простоя.

2. Выбранный подход и описание решения:
Для автоматической группировки рапортов без предварительной ручной разметки применен алгоритм кластеризации текстов на основе методов обработки естественного языка (NLP) и алгоритма машинного обучения K-Means (K-средних).

Пайплайн анализа включает в себя следующие шаги:
1) Предварительная обработка текста: Тексты рапортов переводятся в нижний регистр, очищаются от знаков препинания и спецсимволов. Для решения проблемы изменений окончаний слов в русском языке без установки тяжелых лингвистических библиотек реализован алгоритм псевдо-стемминга — обрезка всех слов до первых 5 символов. Это позволяет объединить в один токен слова типа "подшипник", "подшипника", "подшипники" (основа: "подши").
2) Векторизация TF-IDF: Тексты преобразуются в числовые векторы с использованием меры TF-IDF, которая отражает важность каждого слова в конкретном рапорте по сравнению со всей базой записей. Из векторизации исключены общие русские стоп-слова.
3) Интеллектуальная инициализация центроидов: Для обеспечения высокой стабильности алгоритма и четкого деления по смысловым группам применена инициализация центроидов K-Means на основе доменного знания. Были заданы три опорных вектора ключевых слов:
   - Механика (подшипники, конвейеры, ленты)
   - Электрика (перегрев, моторы, двигатели)
   - Автоматика/КИПиА (ошибки датчиков, АСУ ТП, PLC)
4) Кластеризация: Алгоритм K-Means группирует схожие по векторам рапорты вокруг этих трех центроидов.
5) PCA-визуализация: С помощью метода главных компонент (PCA) размерность векторов снижена до 2D для построения визуального scatter-plot графика.

3. Результаты работы:
Алгоритм успешно распределил 15 тестовых рапортов мастеров по трем непересекающимся кластерам:
- Кластер 0 (Механические неисправности): Сюда попали все записи о заклинивании подшипников, люфтах валов и застревании конвейерных лент.
- Кластер 1 (Электрическая часть): Сгруппировал отчеты о перегреве моторов, тепловой защите двигателей и задымлении коробок передач.
- Кластер 2 (Сбои автоматики и датчиков): Объединил рапорты об ошибках PLC, сбоях АСУ ТП, потере связи с датчиками давления и зависании панелей.

Результаты кластеризации и 2D-визуализация  показаны на изображении. Этот инструмент позволяет менеджменту завода мгновенно выявлять наиболее частые причины простоев оборудования на основе текстовых журналов.
